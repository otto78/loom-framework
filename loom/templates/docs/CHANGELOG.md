## Semantic Versioning

```
MAJOR.MINOR.PATCH

MAJOR: Breaking changes (incompatibilità con versioni precedenti)
MINOR: Nuove feature (backward compatible)
PATCH: Bug fix (backward compatible)
```

### Esempi

- `1.0.0 → 1.0.1` — Bug fix
- `1.0.1 → 1.1.0` — Nuova feature
- `1.1.0 → 2.0.0` — Breaking change
