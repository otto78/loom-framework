The Problem

Il Problema

AI-assisted development faces three critical challenges that make agents unreliable and expensive.

Lo sviluppo assistito da AI affronta tre sfide critiche che rendono gli agenti inaffidabili e costosi.

🚨

Context Window Limits

Limiti Finestra Contesto

Agents lose memory when context resets. Long conversations hit token limits. Critical knowledge gets forgotten.

Gli agenti perdono memoria quando il contesto si resetta. Le conversazioni lunghe raggiungono i limiti di token. La conoscenza critica viene dimenticata.

🔀

Multi-Agent Chaos

Caos Multi-Agente

Switching IDEs loses context. Different agents can't share state. Handoffs between sessions fail.

Cambiare IDE perde il contesto. Agenti diversi non possono condividere lo stato. I passaggi tra sessioni falliscono.

💸

Token Waste

Spreco Token

Agents repeat prompts over and over. 90% accuracy per step = 35% success over 10 steps. Complex tasks chain failures exponentially.

Gli agenti ripetono prompt continuamente. 90% accuratezza per step = 35% successo su 10 step. Task complessi accumulano fallimenti esponenzialmente.

3-Level Framework

Framework a 3 Livelli

Directives → Orchestration → Execution.

How It Works: 3-Level Framework

Come Funziona: Framework a 3 Livelli

LLMs are probabilistic — 90% accuracy per step means 35% success over 10 steps. Loom pushes complexity into deterministic code so the agent only makes decisions.

Gli LLM sono probabilistici — 90% di accuratezza per step significa 35% di successo su 10 step. Loom sposta la complessità in codice deterministico.

📋

Level 1 — Directives

Livello 1 — Direttive

directives/*.md — SOPs in natural language. Define what to do, inputs, outputs, edge cases.

directives/*.md — SOP in linguaggio naturale. Definiscono cosa fare, input, output, casi limite.

🤖

Level 2 — Orchestration

Livello 2 — Orchestrazione

The AI agent's job: interpret intent, choose the right directive, collect inputs, call scripts, handle errors.

Il lavoro dell'agente AI: interpretare l'intento, scegliere la direttiva, raccogliere input, chiamare script.

⚙️

Level 3 — Execution

Livello 3 — Esecuzione

execution/*.py — Deterministic scripts. Same input = same output. JSON output, exit codes, no surprises.

execution/*.py — Script deterministici. Stesso input = stesso output. JSON output, exit code, nessuna sorpresa.

User: "Send welcome email to user@example.com" └─ Agent reads directives/send-email.md ← Level 1 └─ Agent collects inputs, chooses template ← Level 2 └─ Calls execution/send_email.py --recipient ... ← Level 3 └─ Script returns {"status": "sent"} ← deterministic └─ Agent replies: "✅ Welcome email sent" ← Level 2
Utente: "Invia email di benvenuto a user@example.com" └─ Agente legge directives/send-email.md ← Livello 1 └─ Agente raccoglie input, sceglie template ← Livello 2 └─ Chiama execution/send_email.py --recipient ... ← Livello 3 └─ Script ritorna {"status": "sent"} ← deterministico └─ Agente risponde: "✅ Email inviata" ← Livello 2
